A. Magical Sticks

我们可以先确定下来，我们一定只对于未进行过拼接的木棍拼接。

学过等差数列的朋友们，对于 n=6n=6n=6 时都有一个常识：（不会有人没学过等差数列吧）

1+6=2+5=3+4=71+6=2+5=3+4=7 1+6=2+5=3+4=7

即对于任意满足 2∣n2\mid n2∣n 的 nnn。都有：

1+n=2+(n−1)=......=n2+(n2+1)=n+11+n=2+(n-1)=......=\frac{n}{2}+(\frac{n}{2}+1)=n+1 1+n=2+(n−1)=......=2n​+(2n​+1)=n+1

所以答案为 n2\frac{n}{2}2n​。

如果 2∤n2\nmid n2∤n 的话，我们可以把问题转化为 n−1n-1n−1。因为这些木棍拼接起来长度都为 nnn。答案为 n−12+1\frac{n-1}{2}+12n−1​+1。

B. Magical Calendar

我们令 kkk 为一个星期的天数（k≤rk\leq rk≤r）。

我们可以把问题转化为当 n≥kn\geq kn≥k 和 n<kn<kn<k 两种情况。

当 n≥kn \geq kn≥k 时，连续 nnn 天一定会出现在至少两个星期里。我们会发现以第一行的任意一个位置作为起点都会有一个不同的形状，且一定相连。因为此时 n≥kn\geq kn≥k。所以一周里天数为 kkk 时的答案就是 kkk，在最后计算答案时可以用等差数列求和公式解决。

当 n>kn>kn>k 时，无论如何不能跨越星期，所以至多一种情况，就是 nnn 天排列在一个星期里。ansansans 加一即可。

signed main() {
	int t;
	t=read();
	while(t--) {
		n=read();r=read();
		
		int ans=0;
		if(n<=r) {
			ans+=(n-1)*n/2;//等差数列求和,n<=k
			ans+=1;//n>k
		}

			ans+=(r+1)*r/2;//等差数列，注意上下比阿奴街
		}
		cout<<ans<<endl;
	} 
	return 0;
}

C. A Cookie for You

这个题还是推理题

首先我们可以想到，第二类人（以下简称 222）做出任何选择不会印象第一类人（以下简称 111）的选择。因为 222 选择数量少的，111 选择数量多的。所以我们的考虑的主体一定是 222。

那么首先我们可以先判断，如果饼干总数比人数少，一定是 No。然后我们再继续。

我们可以考虑如果数量较少的那一类饼干数量大于等于 222 的人数（听起来怪怪的），那么一定 Yes。因为此时 a+b>n+ma+b>n+ma+b>n+m。222 解决完了，一定应付的过来 111。

如果数量较少的那一类饼干数量小于等于 222 的人数，我们要考虑让 111 去配合 222。但是我们可以发现是不可行的。因为为了让 222 去吃原来数量多的那一类，就一定要让 111 把原来多的吃掉很多，知道比原来较少的还要少。这个局面达不到得到，111 的人数够不够多不说，我们可以发现此时的局面还不如一开始，因为现在较少的那一类饼干的数量比之前较少的那一类饼干的数量还要少。所以一定是 No。

由于 YES,NO 的大小写随意，程序就随意啦

D. Grid-00100

我们先来证明 (a+1)2−a2<(a+2)2−(a+1)(a+1)^2-a^2<(a+2)^2-(a+1)(a+1)2−a2<(a+2)2−(a+1)。用平方差公式展开：(a+1)2−a2=(2a−1)⋅1(a+1)^2-a^2=(2a-1)\cdot 1(a+1)2−a2=(2a−1)⋅1，(a+2)2−(a+1)2=(2a+3)⋅1(a+2)^2-(a+1)^2=(2a+3)\cdot 1(a+2)2−(a+1)2=(2a+3)⋅1。得证。

我们先可以对于：

f(A)=(max⁡(R)−min⁡(R))2+(max⁡(C)−min⁡(C))2f(A)=(\max(R)−\min(R))^2+(\max(C)−\min(C))^2 f(A)=(max(R)−min(R))2+(max(C)−min(C))2

这个式子进行一些思考，根据 (a+1)2−a2<(a+2)2−(a+1)(a+1)^2-a^2<(a+2)^2-(a+1)(a+1)2−a2<(a+2)2−(a+1) 的性质，我们可以知道我们要让 max⁡(R),min⁡(R)\max(R),\min(R)max(R),min(R) 尽量相近，max⁡(C),min⁡(C)\max(C),\min(C)max(C),min(C) 尽量相近。我们就可以用一个类似于“抽屉原理”的思路来解决本题。

我们可以先把矩阵全部用 111 填上，在一点一点把多余的 111 扣掉。

我们在扣 111 的时候。可以考虑选择行，列各不同的 nnn 进行一次去除。如果我们还需去除的不足 nnn 个，就选择剩下的 kkk 个去掉即可，此时答案为 222。如果正好 k=nk=nk=n,那么答案为 000。
